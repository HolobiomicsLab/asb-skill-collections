# count-data-normalization-library-size-correction

## Summary

Normalize RNA-seq read count matrices by accounting for library size and composition bias using trimmed mean of M-values (TMM) normalization, then calculate RPKM values to enable cross-sample comparison of gene expression levels. This skill is essential when raw HTSeq counts reflect different sequencing depths and compositional effects that would otherwise confound biological signals.

## When to use

Apply this skill after generating raw read count matrices from HTSeq (or equivalent gene-level counting tool) and before statistical testing or cross-condition expression comparison. Specifically use it when you have paired-end RNA-seq data aligned to a reference genome with annotated gene features, and you need to quantify expression in biologically comparable units (RPKM) that account for both library size variation and RNA composition bias between samples.

## When NOT to use

- Input is already a processed feature table or pre-normalized expression matrix (e.g., from DESeq2 or Cufflinks); applying TMM normalization twice will compound bias.
- Raw counts were not generated by a union-mode, strand-aware transcript counter; TMM assumes HTSeq-like non-overlapping feature counts.
- Sample metadata or batch information is unavailable; normalization without accounting for known batch effects may remove biological signal.

## Inputs

- Raw read count matrix from HTSeq (rows = genes, columns = samples; counts per annotated feature)
- Gene annotation table with gene identifiers and feature lengths in kilobases
- Sample metadata indicating treatment condition (baicalin-supplemented vs. control)

## Outputs

- Normalization factors per sample (output of edgeR's calcNormFactors())
- RPKM expression table with normalized, length-adjusted expression values (rows = genes, columns = samples)
- Annotated differential expression table for target genes (e.g., LbUGT3 with RPKM values and fold-change ratios)

## How to apply

Load raw count data into R and apply edgeR's TMM (trimmed mean of M-values) normalization via calcNormFactors() to compute normalization factors that correct for library size and composition bias. This step produces scaling factors for each sample. Then use edgeR's rpkm() function to calculate RPKM (reads per kilobase per million mapped reads) values, which divide each count by the gene length in kilobases and the library size-adjusted total mapped reads per sample (in millions). The resulting RPKM table allows direct comparison of expression levels across baicalin-treated and control conditions by normalizing out technical variation. Verify the normalization by checking that RPKM distributions are comparable across samples (e.g., via boxplots or density plots) and that the two-fold expression change is reproducible in the normalized data.

## Related tools

- **HTSeq** (Generates raw read count matrices by counting aligned reads per annotated gene feature using union-mode, reverse-strand settings)
- **edgeR** (Performs TMM normalization via calcNormFactors() and calculates RPKM-normalized expression values using the rpkm() function)
- **R** (Statistical computing environment in which edgeR normalization and RPKM calculation workflows are executed)

## Examples

```
library(edgeR); counts <- read.table('htseq_counts.txt', row.names=1); y <- DGEList(counts); y <- calcNormFactors(y, method='TMM'); rpkm_table <- rpkm(y, gene.length=gene_lengths); write.table(rpkm_table, 'normalized_rpkm.txt', sep='\t')
```

## Evaluation signals

- RPKM values for each gene are positive and comparable across baicalin-treated and control samples without extreme outliers or skew
- Boxplots or density plots of RPKM distributions show similar median and spread across normalized samples, confirming successful library size and composition correction
- LbUGT3 RPKM values in baicalin-supplemented samples are approximately two-fold higher than in control samples, reproducing the expected biological finding
- Normalization factors computed by calcNormFactors() are close to 1.0 for most samples; factors >1.5 or <0.5 may indicate severe composition bias requiring investigation
- MA plot (M = log2 fold-change vs. A = average log2 expression) shows centered distribution around M=0 across the expression range, indicating unbiased normalization

## Limitations

- TMM normalization assumes that most genes are not differentially expressed between samples; if the majority of genes are up- or down-regulated, TMM may over-correct or under-correct normalization factors.
- RPKM values are composition-dependent: if a small number of highly expressed genes dominate total counts, their presence or absence affects RPKM scaling. Consider alternative metrics (e.g., TPM, CPM) in datasets with extreme expression skew.
- Gene length estimates must be accurate and consistent with the reference annotation used in alignment and counting; mismatches (e.g., using different GTF versions) can inflate or deflate RPKM values.
- The RNAseq analysis provided us with a clue for target prioritization, but it will not be helpful when multiple genes show high level of expression or target enzyme is a housekeeping one.
