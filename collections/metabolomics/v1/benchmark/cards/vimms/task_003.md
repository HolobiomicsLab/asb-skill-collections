1. Generate 100 virtual chemicals by sampling m/z uniformly between 100–500 Da using UniformMZFormulaSampler and ChemicalMixtureCreator, requesting both MS1 and MS2 levels. 2. Instantiate an IndependentMassSpectrometer in positive-ion mode with the generated chemical list. 3. Configure a TopNController with N=3 fragments per survey scan, 1 Da isolation window, 10 ppm m/z tolerance, 15 s retention-time tolerance, and 1.75E5 minimum MS1 intensity threshold. 4. Construct an Environment orchestrator spanning 0–1440 s, attach both the mass spectrometer and controller, and invoke env.run() to execute the fixed simulation control loop. 5. Verify the resulting scans list is non-empty and contains both MS1 and MS2 scan records.

## Domain Knowledge
- ViMMS Environment is the orchestrator that synchronizes the mass spectrometer's scan generation with the controller's fragmentation decisions in a fixed three-stage loop: each MS1 scan triggers controller selection of ions, which the spectrometer fragments and returns as MS2 scans.
- TopNController parameters (N, isolation_width, mz_tol, rt_tol, min_ms1_intensity) directly control which and how many precursor ions are selected for fragmentation in each cycle.
- The fixed control-loop architecture (Chemicals → IndependentMassSpectrometer → Controller → Environment.run()) is the conceptual foundation; individual run artifacts (scans, mzML files, evaluation data) are ephemeral outputs of this loop, not its definition.
- Polarity (positive or negative) must match consistently across chemical ionization, mass spectrometer mode, and controller configuration to ensure valid simulation.
