## Workflow Description
1. Sample 100 chemical formulas from HMDB database within m/z range 100–1000 using DatabaseFormulaSampler. 2. Generate KnownChemical objects with retention times, intensities, and MS1 chromatograms using ChemicalMixtureCreator with ms_levels=2. 3. Instantiate IndependentMassSpectrometer with polarity='positive' and the sampled chemicals list. 4. Configure FullScanController with polarity='positive' to acquire only MS1 scans without fragmentation. 5. Create an Environment with the mass spectrometer, controller, min_time=0, max_time=1200, and enable save_eval=True. 6. Execute env.run() to simulate the LC-MS acquisition loop. 7. Write the generated scans to mzML output using Environment.write_mzML().

## Domain Knowledge
- HMDB provides curated metabolite structures with mass and formula annotations; sampling via DatabaseFormulaSampler filters compounds by m/z range to match instrument capabilities.
- FullScanController performs MS1-only acquisition without data-dependent MS2 fragmentation, useful for generating reference full-scan datasets and testing intensity-based methods.
- IndependentMassSpectrometer simulates ion suppression, chromatographic peak shape, and m/z calibration; polarity='positive' assumes protonated molecular ions [M+H]+ as the dominant ionization mode.
- ChemicalMixtureCreator auto-generates retention times (RT), relative intensities, and Gaussian or empirical chromatographic peaks; ms_levels=2 reserves MS2 capability even if FullScanController does not trigger fragmentation.
- Environment.write_mzML() exports scans in the mzML exchange format; save_eval=True persists an EvaluationData pickle containing chemical ground truth and acquisition metadata for downstream benchmarking.
