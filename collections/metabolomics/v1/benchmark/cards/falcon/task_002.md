## Domain Knowledge
- Nearest neighbor indexes trade construction time for query speed by organizing vectors in space to enable rapid similarity lookups without exhaustive comparisons.
- Feature-hashed vectors reduce spectrum dimensionality while preserving spectral similarity relationships needed for clustering.
- Index serialization format must support fast deserialization and efficient neighbor lookup during downstream sparse distance matrix construction.
- The index structure is an intermediate data structure in a four-stage pipeline: hashing → indexing → sparse distance matrix → density-based clustering.
