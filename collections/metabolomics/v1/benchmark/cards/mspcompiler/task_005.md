## Domain Knowledge
- MSP (NIST MS Search) format encodes mass spectral metadata (precursor m/z, fragment peaks, compound name, SMILES, InChIKey) as newline-delimited text blocks separated by blank lines; read_multilibs() must correctly parse variable schema across batch-processed or in-house files.
- The future::plan() function sets the evaluation strategy for subsequent parallelizable operations; plan(multisession(workers=N)) spawns N background R processes sharing memory via the multisession strategy, distinct from other backends (socket, cluster, fork).
- detectCores() returns the total logical CPU cores available; reserving one core (detectCores() - 1) for the main R process prevents system overload and is standard practice for interactive multi-worker configurations.
- Merging multiple spectral library objects via c() in R concatenates their entries; serial equivalence testing (comparing parallel-processed output to sequential read_multilibs() on the same input directory) is the canonical validation approach in mspcompiler workflows.
