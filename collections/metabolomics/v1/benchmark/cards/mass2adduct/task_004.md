## Domain Knowledge
- Cardinal version 2.2 or later is required for MSProcessedImagingExperiment and MSContinuousImagingExperiment interoperability with mass2adduct.
- The msimat format requires a numeric matrix with m/z values as column names and pixels as row names, with intensity values as entries; Cardinal conversion must produce this structure.
- The massdiff() function calculates mass differences for all possible ion pairs; the resulting object has three columns (A: parent m/z, B: adduct m/z, diff: mass difference).
- adductMatch() identifies the closest-matching adduct in the reference database for each mass difference, accounting for the fact that mass resolution varies with m/z.
- The built-in reference datasets adducts and adducts2 contain biologically-relevant chemical species; users may substitute custom tables with three columns (name, formula, mass).
