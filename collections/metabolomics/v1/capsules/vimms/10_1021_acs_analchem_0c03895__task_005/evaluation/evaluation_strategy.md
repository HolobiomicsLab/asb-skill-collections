# Evaluation Strategy

## Direct Checks

- verify file exists: EvaluationData pickle artifact output from WeightedDEWController run
- verify file exists: mzML output file generated by WeightedDEWController simulation
- verify file_format_is: output mzML file conforms to mzML schema
- script_runs: ViMMS simulation loop with WeightedDEWController instantiation and use_weighteddew_exclusion parameter toggled to True, robust to parameter choices in isolation_width and N equivalents
- output_matches_reference: evaluation metrics (e.g. fragmentation coverage, number of MS2 scans acquired) differ numerically from baseline TopNController run — no canonical answer, multiple defensible approaches to quantify difference (percent change, absolute difference, statistical test)

## Expert Review

- Confirm that WeightedDEWController behavior with use_weighteddew_exclusion=True produces chemically meaningful differences in controller decision logic compared to TopNController baseline
- Assess whether observed metric differences are attributable to controller strategy change rather than random simulation variance
