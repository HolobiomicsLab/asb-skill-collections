## Research Question
Do transcript-level estimates obtained via tximport with txOut=TRUE produce identical gene-level count matrices after summarization with summarizeToGene as compared to direct gene-level import using txOut=FALSE?

## Connected Finding
Gene-level count matrices derived from transcript-level tximport output (txOut=TRUE) followed by summarizeToGene are identical to those produced by direct gene-level tximport (txOut=FALSE), as verified by all.equal() comparison.

## Task Description
Run tximport on salmon transcript-abundance files with txOut=TRUE to obtain transcript-level counts, then apply summarizeToGene with a tx2gene mapping to collapse to gene level, and verify the resulting gene-level count matrix is identical to direct import with txOut=FALSE.

## Domain Knowledge
- The txOut parameter controls whether tximport returns transcript-level (txOut=TRUE) or gene-level (txOut=FALSE, default) matrices; both workflows should produce identical gene-level counts when summarizeToGene is applied downstream.
- A tx2gene data.frame must contain at minimum two columns: transcript identifiers (matching the row names of the quantification output) and corresponding gene identifiers for proper aggregation.
- Gene-level summarization collapses transcript-level abundance, counts, and length estimates; the length matrix is particularly important as it accounts for differential isoform usage across samples.
- Salmon quantification files (quant.sf) contain per-transcript estimates of abundance (TPM), estimated counts, and effective transcript length; tximport reads and structures these into matrix form.
