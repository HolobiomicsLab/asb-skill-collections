## Research Question
When DESeq2::DESeqDataSetFromTximport is called with a tximport output object (txi) and a sample table, does the resulting DESeqDataSet correctly embed the transcript-length-derived offset matrix needed for gene-level differential expression analysis using the 'original counts and offset' method?

## Domain Knowledge
- Transcript-level quantification outputs (abundance, counts, length) must be aggregated to gene-level matrices for count-based differential expression analysis.
- The offset matrix encodes effective gene length variation across samples, correcting for differential isoform usage that would otherwise bias gene-level count estimates.
- DESeq2::DESeqDataSetFromTximport automatically incorporates the length matrix as an offset term in the GLM, eliminating the need for manual matrix construction.
- The tximport length matrix is required as input to DESeqDataSetFromTximport; missing or malformed length data will cause offset embedding to fail.
