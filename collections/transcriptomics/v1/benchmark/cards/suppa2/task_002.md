## Domain Knowledge
- PSI values represent the percent spliced in for an alternative splicing event, calculated as the ratio of transcripts including the alternative exon(s) to the total abundance of all isoforms in that event.
- ioe (isoform-to-event) and ioi (isoform-overlap-incidence) are SUPPA2 standard event definition file formats that encode how transcript isoforms map to specific alternative splicing events.
- Transcript quantification tools like Salmon and kallisto output transcript-level abundance estimates (TPM or estimated counts) that must be mapped to events via the event definition file.
- PSI values range from 0 (no inclusion) to 1 (full inclusion); missing or ambiguous isoforms may result in NA or zero values depending on how SUPPA2 handles unmapped transcripts.

## Methodology Summary
1. Load transcript quantification output with transcript identifiers and abundance values (TPM or counts).
2. Parse SUPPA2 event definition file (ioe or ioi) to extract event–transcript mappings.
3. Aggregate transcript abundances per event according to inclusion/exclusion isoform sets.
4. Calculate PSI as the ratio of inclusion isoform total to total event abundance.
5. Validation: PSI matrix has one row per event, one column per sample, all values in [0,1].
