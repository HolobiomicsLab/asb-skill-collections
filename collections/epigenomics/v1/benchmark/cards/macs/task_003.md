## Domain Knowledge
- In paired-end ChIP-Seq, the fragment length d is the average distance between the 5' mapping positions of read pairs and represents the actual DNA fragment bound by the protein; this value is critical for accurate peak calling.
- The predictd subcommand reports average insertion length (fragment length) by analyzing the mapping locations of read pairs; this reported value should be recorded for use in downstream normalization and peak-calling steps.
- For paired-end data in BEDPE format, the macs3 pileup function automatically extends signal across the entire region between the two 5' mapping positions without requiring an explicit --extsize parameter.
- Narrow peaks are defined by stronger, more localized enrichment signals and are output in narrowPeak format, which is a specialized BED variant that includes the summit position (peak maximum) in the final column.
- The local Poisson test in MACS3 compares ChIP signal at each base pair against a local lambda (background expectation) computed from control data, accounting for local genomic biases at multiple scales (d, slocal, llocal, and genome-wide background).

## Methodology Summary
1. Estimate average insertion length (fragment length d) from paired-end read pairs using macs3 predictd with -f BEDPE flag.
2. Generate ChIP signal pileup track using macs3 pileup with paired-end input (no --extsize needed); pileup automatically spans between read pair 5' positions.
3. Build local background noise tracks from control data at multiple scales (d, slocal=1kb, llocal=10kb) using macs3 pileup with -B option and compute genome-wide background.
4. Normalize and combine background tracks using macs3 bdgopt and macs3 bdgcmp to generate maximum local bias (lambda) track scaled to match ChIP sequencing depth.
5. Compare ChIP pileup to local lambda using macs3 bdgcmp with Poisson model (-m qpois) to generate q-value score track at each genomic position.
6. Call narrow peaks by identifying regions exceeding a q-value cutoff using macs3 bdgpeakcall with read length (-g) and fragment length (-l) parameters; output narrowPeak format with summit coordinates.
7. Validation: confirm narrowPeak file exists, contains proper BED format with 5 required columns plus summit position, and reported average insertion length from predictd matches expected ~253 bp metric.
