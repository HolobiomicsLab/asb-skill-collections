## Domain Knowledge
- BAM files must be coordinate-sorted (by genome position) prior to fragment file conversion to ensure efficient sequential processing.
- Fragment files store paired-end read information as intervals with associated cell barcodes and are typically compressed using gzip or zstandard for storage efficiency.
- pp.make_fragment_file by default treats fragment files as paired-end; single-end data requires explicit specification via the is_paired=False parameter.
- 10X-derived BAM files contain cell barcode and quality information in tags (e.g., CB, CR); pp.make_fragment_file supports 10X BAM parsing when source='10x' is specified.
